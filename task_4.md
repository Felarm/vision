
RED_STATS


Для отправки сообщения о том, что у такого-то клиента пришла картинка с таким-то количеством красного цвета можно отправлять json объект в POST запросе
POST /info
{
    "client_id": "integer",
    "red_color_percentage": "float"
}


Для подписки  на оповещения клиент может использовать следующий запрос
POST /subscribe
{
    "email": "string",
    "client_id": "integer"
}


Для получения статистики по времени клиент может использовать следующий запрос
GET /stats
{
    "start_time": "string", (начало временного прмоежутка)
    "end_time" : "string", (конец временного промежутка)
    "red_tol" : "float" (заданного значение красного цвета на изображениях)
}'''


Для хранения данных лучше использовать NoSQL базу данных, например MongoDB.
Если использовать SQL БД, то для масшитабирования нужен будет дополнительный HDD и больше времени на прокидывание связей между таблицами.
Для масштабирования с MongoDB можно использовать дополнительный сервер с установленной MongoDB, который будет проще сконфигурировать, чтобы БД работала на нескольких серверах, как на одном.
